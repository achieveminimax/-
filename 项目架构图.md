### 模块说明

| 模块 | 类型 | 职责 |
|------|------|------|
| seckill-parent | Parent POM | 统一依赖管理、版本控制 |
| seckill-infrastructure | Jar | Redis、RabbitMQ、MyBatis-Plus 配置 |
| seckill-common | Jar | 统一响应、异常处理、工具类、常量 |
| seckill-user | Jar | 用户注册、登录、JWT 认证、个人信息 |
| seckill-goods | Jar | 商品管理、分类管理、库存管理 |
| seckill-seckill | Jar | 秒杀核心：限流、预扣库存、异步下单 |
| seckill-order | Jar | 订单管理、支付模拟、超时取消 |
| seckill-admin | Jar | 管理后台、数据统计、权限控制 |
| seckill-application | Jar | Spring Boot 启动入口，聚合所有模块 |

## 5. 秒杀核心流程

```mermaid
sequenceDiagram
    autonumber
    participant C as 客户端
    participant API as API 网关
    participant Limiter as 限流器<br/>Redis+Lua
    participant Seckill as 秒杀服务
    participant Redis as Redis
    participant MQ as RabbitMQ
    participant Order as 订单服务
    participant DB as MySQL

    C->>API: 1. 请求秒杀接口<br/>POST /api/seckill/do
    API->>Limiter: 2. 限流检查<br/>令牌桶算法
    alt 限流通过
        Limiter-->>API: 允许通过
        API->>Seckill: 3. 执行秒杀逻辑
        
        Seckill->>Redis: 4. 检查用户是否已秒杀<br/>SETNX seckill:user:{userId}:{goodsId}
        alt 已秒杀
            Redis-->>Seckill: 已存在
            Seckill-->>API: 返回错误：已参与秒杀
        else 未秒杀
            Redis-->>Seckill: 设置成功
            Seckill->>Redis: 5. Lua 脚本扣减库存<br/>DECR seckill:stock:{goodsId}
            alt 库存不足
                Redis-->>Seckill: 库存 < 0
                Seckill->>Redis: 删除用户已秒杀标记
                Seckill-->>API: 返回错误：库存不足
            else 扣减成功
                Redis-->>Seckill: 扣减成功
                Seckill->>MQ: 6. 发送异步下单消息
                MQ-->>Seckill: 消息确认
                Seckill-->>API: 返回：排队中
                API-->>C: 响应：排队中，orderNo
            end
        end
    else 限流拒绝
        Limiter-->>API: 拒绝请求
        API-->>C: 返回 429 Too Many Requests
    end

    MQ->>Order: 7. 消费下单消息
    Order->>DB: 8. 创建订单记录
    Order->>DB: 9. 数据库扣减库存<br/>乐观锁 UPDATE
    alt 数据库扣减成功
        Order->>Redis: 10. 更新订单状态缓存
        Order->>MQ: 11. 发送延迟消息<br/>TTL=15分钟
    else 数据库扣减失败
        Order->>Redis: 回滚 Redis 库存<br/>INCR
        Order->>Redis: 删除用户已秒杀标记
    end

    Note over MQ: 15分钟后
    MQ->>Order: 12. 消费延迟消息
    Order->>DB: 13. 查询订单状态
    alt 未支付
        Order->>DB: 取消订单
        Order->>Redis: 回滚库存
        Order->>Redis: 删除用户已秒杀标记
    else 已支付
        Order-->>MQ: 忽略消息
    end
```

```mermaid
flowchart TB

        DB_ORDER[("订单表<br/>t_order")]
    end

    WEB --> CONTROLLER
    CONTROLLER --> SERVICE
    SERVICE --> REDIS_STOCK
    SERVICE --> REDIS_USER
    SERVICE --> MQ_PRODUCER
    MQ_PRODUCER --> EXCHANGE
    EXCHANGE --> QUEUE_ORDER
    QUEUE_ORDER --> MQ_CONSUMER
    MQ_CONSUMER --> DB_ORDER
    MQ_CONSUMER --> DB_SECKILL
    MQ_PRODUCER --> QUEUE_DELAY
    QUEUE_DELAY -->|TTL到期| DLX
    DLX --> MQ_CONSUMER
    SERVICE --> REDIS_GOODS
    SERVICE --> DB_USER
    SERVICE --> DB_GOODS
    SERVICE --> DB_SECKILL
```

### 部署说明

| 层级 | 组件 | 部署方式 | 高可用方案 |
|------|------|----------|------------|
| 网络层 | DNS | 云解析 | 多线路解析 |
| 网络层 | CDN | 云 CDN | 多节点加速 |
| 网关层 | Nginx | 多台部署 | Keepalived + VIP |
| 应用层 | Spring Boot | Docker 容器 | 多实例负载均衡 |
| 缓存层 | Redis | Docker / 云托管 | 主从 + 哨兵模式 |
| 消息层 | RabbitMQ | Docker 集群 | 镜像队列集群 |
| 数据库 | MySQL | Docker / 云托管 | 主从复制 + 读写分离 |
