## 1. 设计概述

### 1.1 设计目标

| 目标 | 描述 |
|------|------|
| 用户体验 | 简洁直观、操作流畅、信息层次清晰 |
| 视觉风格 | 现代简约、科技感、电商属性 |
| 品牌调性 | 年轻活力、可信赖、高效率 |
| 秒杀氛围 | 紧迫感、热度感、抢购刺激 |

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **一致性** | 相同功能使用相同的交互方式和视觉样式 |
| **反馈性** | 用户操作后立即给予明确反馈 |
| **容错性** | 防止用户误操作，提供撤销机制 |
| **高效性** | 减少用户操作步骤，核心功能快速触达 |
| **可访问性** | 符合 WCAG 2.1 标准，支持无障碍访问 |

### 1.3 用户画像

#### C端用户（普通消费者）

| 属性 | 描述 |
|------|------|
| 年龄 | 18-35 岁 |
| 特征 | 追求性价比、喜欢参与促销活动、移动端为主 |
| 需求 | 快速找到秒杀商品、流畅抢购体验、订单状态实时更新 |
| 痛点 | 秒杀抢不到、页面卡顿、支付流程繁琐 |

#### B端用户（管理员）

| 属性 | 描述 |
|------|------|
| 年龄 | 25-40 岁 |
| 特征 | 熟悉后台操作、关注数据指标、需要高效管理工具 |
| 需求 | 快速创建秒杀活动、实时监控数据、批量操作 |
| 痛点 | 数据统计不直观、操作流程繁琐 |

## 2. 设计规范

### 2.1 色彩系统

#### 2.1.1 主色调

| 颜色名称 | 色值 | 用途 |
|---------|------|------|
| **品牌主色** | `#FF4D4F` | 主要按钮、重要信息、秒杀价格、促销标签 |
| **品牌辅色** | `#FF7A45` | 悬停状态、渐变过渡、次要强调 |
| **品牌浅色** | `#FFF1F0` | 主色背景、选中状态背景、标签背景 |

#### 2.1.2 功能色

| 颜色名称 | 色值 | 用途 |
|---------|------|------|
| **成功色** | `#52C41A` | 成功提示、已完成状态、支付成功 |
| **警告色** | `#FAAD14` | 警告提示、待处理状态、倒计时 |
| **错误色** | `#FF4D4F` | 错误提示、失败状态、必填标识 |
| **信息色** | `#1890FF` | 信息提示、链接、帮助说明 |

#### 2.1.3 中性色

| 颜色名称 | 色值 | 用途 |
|---------|------|------|
| **标题色** | `#262626` | 页面标题、重要文字 |
| **正文色** | `#595959` | 正文内容、描述文字 |
| **辅助色** | `#8C8C8C` | 辅助信息、占位文字、禁用文字 |
| **边框色** | `#D9D9D9` | 边框、分割线 |
| **背景色** | `#F5F5F5` | 页面背景、卡片背景 |
| **纯白色** | `#FFFFFF` | 卡片背景、输入框背景 |

### 2.2 字体规范

#### 2.2.1 字体家族

```
/* 中文字体 */
font-family: -apple-system, BlinkMacSystemFont, 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', sans-serif;

/* 英文/数字字体 */
font-family: 'SF Pro Display', 'Helvetica Neue', Arial, sans-serif;

/* 等宽字体（订单号、验证码等） */
font-family: 'SF Mono', 'Consolas', 'Monaco', monospace;
```

#### 2.2.2 字号层级

| 层级 | 字号 | 字重 | 行高 | 用途 |
|------|------|------|------|------|
| H1 | 28px | 600 | 1.3 | 页面主标题 |
| H2 | 24px | 600 | 1.35 | 模块标题 |
| H3 | 20px | 500 | 1.4 | 卡片标题、弹窗标题 |
| H4 | 18px | 500 | 1.45 | 列表标题 |
| Body | 14px | 400 | 1.5 | 正文内容 |
| Caption | 12px | 400 | 1.5 | 辅助信息、时间戳 |
| Small | 10px | 400 | 1.5 | 标签、角标 |

### 2.3 间距系统

采用 **8px 基准网格** 系统：

| 间距变量 | 数值 | 用途 |
|---------|------|------|
| xs | 4px | 紧凑间距、图标与文字间距 |
| sm | 8px | 元素内间距、列表项间距 |
| md | 16px | 模块间距、卡片内边距 |
| lg | 24px | 区块间距、页面边距 |
| xl | 32px | 大区块间距 |
| xxl | 48px | 页面顶部间距 |

### 2.4 圆角规范

| 圆角变量 | 数值 | 用途 |
|---------|------|------|
| sm | 4px | 小按钮、标签、徽章 |

| xl | 16px | 底部弹窗、大容器 |

### 2.5 阴影规范

| 阴影变量 | 数值 | 用途 |
|---------|------|------|
| sm | `0 1px 2px rgba(0,0,0,0.05)` | 卡片悬浮 |

| lg | `0 8px 24px rgba(0,0,0,0.15)` | 模态框、重要提示 |

### 2.6 图标规范

| 属性 | 规范 |
|------|------|

#### 4.1.3 交互设计

| 交互场景 | 行为 |
|---------|------|
| 输入框获得焦点 | 边框变为主色调，显示输入提示 |
| 输入框失去焦点 | 校验输入格式，错误显示红色边框 + 错误提示 |
| 点击登录 | 按钮显示 Loading 状态，禁用重复点击 |
| 登录成功 | Toast 提示"登录成功"，跳转至首页 |
| 登录失败 | Toast 提示错误信息，输入框抖动 |
| 连续失败 5 次 | 弹窗提示"账号已锁定 30 分钟" |

#### 4.1.4 状态设计

| 状态 | 视觉表现 |
|------|---------|
| 默认 | 输入框灰色边框，按钮可点击 |
| 输入中 | 输入框主色边框，按钮可点击 |
| 校验错误 | 输入框红色边框，下方显示错误文字 |
| 提交中 | 按钮 Loading 动画，禁用状态 |
| 禁用 | 按钮灰色，不可点击 |

## 6. 交互设计

### 6.1 全局交互规范

#### 6.1.1 加载状态

| 场景 | 加载方式 |
|------|---------|
| 页面初始化 | 全屏 Loading 动画 + 骨架屏 |
| 列表加载 | 底部加载更多动画 |
| 按钮操作 | 按钮 Loading 状态 |
| 数据刷新 | 下拉刷新动画 |

#### 6.1.2 反馈提示

| 类型 | 样式 | 持续时间 |
|------|------|---------|
| 成功提示 | Toast 绿色图标 | 2s 自动消失 |
| 错误提示 | Toast 红色图标 | 3s 自动消失 |
| 警告提示 | Toast 橙色图标 | 2.5s 自动消失 |
| 信息提示 | Toast 蓝色图标 | 2s 自动消失 |
| 确认弹窗 | Modal 弹窗 | 需用户操作关闭 |

#### 6.1.3 手势操作

| 手势 | 场景 | 行为 |
|------|------|------|
| 下拉 | 列表页 | 刷新数据 |
| 上拉 | 列表页 | 加载更多 |
| 左滑 | 订单卡片 | 显示删除按钮 |
| 点击 | 按钮/链接 | 触发操作 |
| 长按 | 图片 | 保存图片 |

### 6.2 秒杀核心交互

#### 6.2.1 秒杀按钮状态

| 状态 | 视觉 | 行为 |
|------|------|------|
| 未开始 | 灰色"未开始" | 显示开始时间，点击设置提醒 |
| 进行中-可抢 | 红色"立即抢购" | 点击触发秒杀流程 |
| 进行中-已购 | 灰色"已购买" | 点击跳转订单详情 |
| 已结束 | 灰色"已结束" | 不可点击 |
| 已售罄 | 灰色"已售罄" | 不可点击 |
| 抢购中 | 红色+Loading | 禁用，显示"排队中..." |

#### 6.2.2 秒杀排队交互

```
用户点击"立即抢购"
    │
    ▼
按钮变为 Loading 状态
    │
    ▼
弹窗显示"正在排队..."
    │
    ├── 成功 → 跳转结果页（成功）
    │
    └── 失败 → 弹窗提示失败原因
              │
              ├── 售罄 → "商品已售罄"
              ├── 已购 → "您已购买过该商品"
              └── 其他 → "系统繁忙，请稍后重试"
```

### 6.3 表单交互规范

#### 6.3.1 输入校验

| 校验时机 | 方式 |
|---------|------|
| 输入时 | 实时校验格式（如手机号、密码强度） |
| 失焦时 | 校验完整性和唯一性 |
| 提交时 | 校验所有字段，高亮错误项 |

#### 6.3.2 错误提示

| 类型 | 样式 |
|------|------|
| 格式错误 | 输入框下方红色文字提示 |
| 必填为空 | 输入框红色边框 + "此项为必填" |
| 唯一性冲突 | Toast 提示 "用户名已存在" |

## 7. 响应式设计

### 7.1 断点设计

| 设备类型 | 断点 | 布局方式 |
|---------|------|---------|
| 手机竖屏 | < 576px | 单列布局，底部导航 |
| 手机横屏/小平板 | 576px - 768px | 单列布局，底部导航 |
| 平板 | 768px - 992px | 双列布局，侧边导航 |
| 桌面 | 992px - 1200px | 多列布局，侧边导航 |
| 大屏 | > 1200px | 固定宽度居中，侧边导航 |

### 7.2 适配策略

#### 7.2.1 用户端（移动优先）

| 组件 | 手机 | 平板 | 桌面 |
|------|------|------|------|
| 商品列表 | 单列卡片 | 双列卡片 | 四列卡片 |
| 秒杀卡片 | 横向滑动 | 网格布局 | 网格布局 |
| 底部导航 | 固定底部 | 固定底部 | 隐藏，顶部导航 |
| 侧边栏 | 无 | 无 | 显示 |

#### 7.2.2 管理后台（桌面优先）

| 组件 | 桌面 | 平板 | 手机 |
|------|------|------|------|
| 侧边导航 | 固定展开 | 可折叠 | 抽屉式 |
| 表格 | 完整显示 | 横向滚动 | 卡片列表 |
| 统计图表 | 多列并排 | 两列 | 单列堆叠 |
